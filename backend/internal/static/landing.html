StS2 Multiplayer Statistics

Slay the Spire 2 のマルチプレイ／シングルプレイのラン統計を mod で収集し、URL を共有して仲間と閲覧できるサービス。

Status: 公開中 無料 / 個人運用 GitHub

このページは何？

このサーバ自体（https://sts2stats.fly.dev）は API バックエンドです。

ラン統計を見るには、mod が発行する 共有 URL（/s/<uuid> 形式）を開く必要があります。

共有 URL は mod がゲーム内でラン開始時に 自動でクリップボードにコピー します。 マルチプレイならホストの友達からその URL をもらってください。

使い方

1. mod を入れる

GitHub Releases から StsStats.dll と StsStats.json をダウンロードし、Slay the Spire 2 の mods フォルダに置きます。 

Slay the Spire 2/...../mods/StsStats/
  ├── StsStats.dll
  └── StsStats.json

2. ゲームを起動してラン開始

mod が以下を自動で行います:

  • このサーバにセッションを作成
  • 共有 URL をクリップボードにコピー
  • ローカルログ（sts_stats.jsonl）にも URL を記録

3. ブラウザで共有 URL を開く

クリップボードを貼り付けるだけ。戦闘中はリアルタイム（10秒ポーリング）で更新されます。

マルチプレイ

ホスト 1 人だけが mod を入れていれば全員ぶんの統計が記録されます。 Slay the Spire 2 は決定論的ロックステップ方式で、全クライアントが同じイベントを処理しているためです。

共有 URL を Discord 等で渡せば、仲間は mod を入れずに閲覧だけできます。

送信されるデータ

項目内容
プレイヤー識別Steam ID と Steam 表示名
ラン情報キャラクター・Ascension・seed
戦闘統計ターンごとの与/被ダメ、ブロック、カード使用、デバフ付与
結果戦闘の勝敗、ラン全体の勝敗・最終階層

詳細仕様は API 仕様 参照。 送信を完全に止めたい場合は mod を削除するか、config.json でセルフホストに切り替えてください。

プライバシー・セキュリティ

  • 共有 URL を知っている人は誰でも統計を 閲覧 できます（書き込みは不可）
  • 書き込みトークンは mod のローカルにのみ保存され、URL には含まれません
  • セッションデータは予告なく削除する場合があります（個人運用のため）
  • サーバの可用性は保証しません。バックアップは mod のローカルログにあります

セルフホストしたい

公開サーバを使いたくない場合、自分で立てて mod の接続先を切り替えられます。詳細は GitHub の README 参照。

# 1. リポジトリを clone してビルド
make docker-run

# 2. mod の接続先を localhost に向ける
make mod-use-local
